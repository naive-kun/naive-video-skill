# Migration: No State To 1.0

## When

Use when a project was created with the original skill and has caption, preview, or final files but no `.naive-video-state.json`.

## Steps

1. Detect the main video from the user's current task or ask for it.
2. Run `python3 tools/bootstrap.py --project <project_dir> --video <main_video> --adopt-existing`.
3. Detect existing SRT, CSV, preview, and final outputs without moving them.
4. Create missing project-memory templates, including `VIDEO_RETRO.md`, without overwriting authored files.
5. Set the stage to the last gate that actually passes.
6. Keep approval pending unless the user confirms a specific preview version was approved.

## Safety

Do not rename, move, or delete existing media. Do not infer persistent style preferences from old renders without user confirmation.
